数据库基本概念

  • 数据库：简称 DB（DataBase）
    存储数据的仓库，实际上就是一堆文件，文件中存储特定格式的数据。
  • 数据库管理系统：简称 DBMS（DataBaseManagementSystem）
    用来管理数据库中的数据，可以对数据进行增删改查（CRUD）
    常见的数据库管理系统：MySQL、Oracle、MS SqlServer、DB2、sybase等等…
  • SQL：结构化查询语言
    我们需要学习 SQL 语句，DBMS 会根据我们编写的 SQL 语句，对数据进行增删改查（CRUD）。

DBMS —> SQL —> DB

  • 表：数据库中用表格表示数据，非常直观~
    学号 姓名 性别
    1234 小明 男
    4321 小红 女

行（row）：被称为数据/记录。
列（column）：被称为字段，字段有字段名、数据类型、约束等属性。

DQL 多表查询

连接查询

将多张表连接起来查询
连接查询分为：内连接、外连接、全连接

语法：select 字段1, 字段2 from 表1, 表2;

笛卡尔积现象

当两张表进行连接查询，在没有任何条件限制时，最终查询结果数 = 两张表行数乘积

消除笛卡尔积

为了数据的有效性，我们要消除笛卡尔积现象

内连接

表之间没有主次关系

  • 等值连接
    连接条件是等量关系

  • 非等值连接
    连接条件不是等量关系

  • 自连接
    一张表看成两张表

外连接

表之间有主次关系：左或右出现哪个，哪个就是主表
主表的数据会被全部查询出来

  • 右外连接（右连接）

  • 左外连接（左连接）

约束

约束（constraint），创建表时对字段进行约束，保证数据的有效性和完整性

  • 非空约束：not null
  • 唯一性约束：unique
  • 主键约束：primary key 主键非空且唯一
  • 外键约束：foreign key
  • 检查约束：check（mysql不支持，oracle支持）

表级约束

以上约束都是列级约束。当多个字段联合起来添加约束时，需要表级约束

  • unique() 多个字段联合唯一
  • 主键可以表级约束
  • not null只能列级约束

主键自增

外键约束

将外键关联另一张表的字段
添加外键可以减少数据的冗余
外键约束，防止外键所关联的字段不存在，即外键值只能是关联字段的值

语法：foreign key (外键名) references 另一张表名(另一张表的字段)
例如

  • 外键可以为 null
  • 外键关联的字段必须为 unique

存储引擎

存储引擎（MySQL特有术语）是一个表存储数据的方式

MyISAM

可被转换为压缩、只读表来节省空间
每个表对应三个文件：

  • 格式文件（frm）
  • 数据文件（MYD）
  • 索引文件（MYI）

主键或 unique约束的字段自动创建索引

InnoDB

默认，重量级引擎
支持事务处理，非常安全
支持数据库崩溃后自动恢复机制
表的内容存储在 InnoDB表空间 tablespace（存储数据+索引）

MEMORY

数据和索引都存储在内存中
优点：查询效率最高，不需要和硬盘交互
缺点：不安全，关机后数据消失

事务

一个事务（transaction）就是一个完整的业务逻辑，是最小的工作单元，不可再分

只有DML语句和事务有关系，其他语句不行

  • select
  • delete
  • update

提交和回滚

InnoDB存储引擎：提供一组记录事务性活动的日志文件
每一条DML操作都会记录到日志文件中，提交或回滚对日志文件进行处理

默认自动提交（每执行一条DML语句，提交一次）
回滚只能回滚到上一次的提交点

  • 提交：commit
  • 回滚：rollback

特性

  • A：原子性
    事务是最小的工作单位，不可再分
  • C：一致性
    同一个事务中，所有操作必须同时成功或失败
  • I：隔离性
    事务A与事务B之间具有一定的隔离
  • D：持久性
    事务结束的保障。事务提交，相当于把没有保存到硬盘上的数据保存到硬盘上

隔离级别

mysql中默认的事务隔离级别为 repeatable read

  • 读未提交：read uncommitted（最低级别）
    事务的数据未提交就可以被读取到
    存在问题：Dirty Read脏读现象，我们称读到了脏数据
  • 读已提交：read committed
    事务的数据提交后才能被读取到
  • 可重复读：repeatable read
    事务开启后，当前事务中读取的数据不变，直到该事务结束
    会有幻象现象
  • 序列化/串行化：serializable（最高级别）
    解决所有问题，但效率最低，事务排队，不能并发

查看隔离级别

  • select @@transaction_isolation;
  • show variables like '%transaction_isolation%'

索引

索引添加在字段上，是提高查询效率的一种机制
索引相当于书的目录，缩小扫描范围

添加索引的条件：

  • 数据量庞大
  • 该字段总是被扫描（经常出现在where后边）
  • 该字段很少的DML操作

不要随意添加索引，太多会降低系统的性能
主键和 unique 自动添加索引，通过主键或unique约束的字段查询，效率较高

数据库设计三范式

数据库设计范式是数据库表的设计依据，可以避免表中数据的冗余，节省空间

第一范式

要求任何一张表必须有主键，每一个字段原子性不可再分
最核心，最重要的范式，所有表的设计都要满足

第二范式

建立在第一范式的基础上
要求所有非主键字段必须依赖主键，不要产生部分依赖

第三范式

建立在第二范式的基础上
要求所有非主键字段必须直接依赖主键，不要产生传递依赖

总结

  • 一对一设计
    一对一，外键唯一
  • 一对多设计
    一对多，两张表，多的表加外键
  • 多对多设计
    多对多，三张表，关系表俩外键

数据库三范式是理论上的

最终的目的都是为了满足客户的需求，有时会拿冗余换执行速度，因为在SQL语句中，表和表之间连接越多，效率越低（笛卡尔积）
并且对于开发人员来说，sql语句编写的难度也会降低
